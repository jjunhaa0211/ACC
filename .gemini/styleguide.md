# Gemini Review Style Guide

## Language
- 모든 PR 리뷰 코멘트, 요약, 피드백은 한국어(ko-KR)로 작성합니다.
- 코드/명령어/식별자는 원문을 유지하되 설명은 한국어로 제공합니다.

## Review Priority
1. 보안/데이터 손상/런타임 장애 가능성
2. 기능 회귀 가능성
3. 테스트 누락 및 품질 리스크
4. 유지보수성 개선 제안

## Output Style
- 핵심 이슈를 먼저 제시하고, 영향 범위와 수정 방향을 짧고 명확하게 작성합니다.
- 불필요한 장문보다는 실행 가능한 항목 중심으로 작성합니다.
